# ⚡ Emil Kowalski - Design Engineering Skill

Este módulo integra los principios de **Design Engineering** de Emil Kowalski. Su objetivo es cerrar la brecha entre el diseño visual y la implementación técnica, asegurando interfaces de alta fidelidad y rendimiento en el **Client** de Catalyst.

---

## 📍 Información de Origen

- **Fuente Original:** [emilkowalski/skill](https://github.com/emilkowalski/skill)
- **Especialidad:** Craftsmanship, micro-interacciones, animaciones refinadas y detalles de implementación "pixel-perfect".

---

## 📖 Aplicación en el Proyecto

Mientras que _Impeccable_ define la estructura y _Taste_ define el estilo, esta skill se encarga de la **ejecución técnica** del frontend:

1.  **Interacciones Refinadas:** Implementación de estados de carga (skeletons), transiciones suaves y feedback táctil/visual en los botones.
2.  **Calidad de Código CSS:** Uso de técnicas modernas de CSS para lograr efectos complejos con el mínimo impacto en rendimiento.
3.  **Detalles de Pulido:** Manejo de sombras, bordes, tipografía dinámica y espaciados con un enfoque de ingeniería de precisión.

---

## 📂 Estructura

- Contiene la definición `SKILL.md` que actúa como el estándar de calidad para cualquier componente interactivo desarrollado en la carpeta `/client`.

---

## 🚦 Instrucciones para el Agente (Claude/Gemini)

- **Implementación:** Al generar componentes (ej. el uploader de OCR o el feed de registros), aplica los estándares de "Design Engineering" de Emil para asegurar que la interacción sea fluida.
- **Micro-interacciones:** Prioriza el uso de animaciones sutiles y estados de transición que mejoren la percepción de velocidad de la aplicación.
- **Revisión de Código:** Antes de finalizar un componente de UI, verifica si cumple con los criterios de pulido definidos en esta skill.
- **Relación con otras Skills:** \* `Impeccable` pone los cimientos.
  - `Taste` define la estética.
  - `Emil Design Eng` asegura que la **construcción técnica** sea de clase mundial.

---

## 📝 Nota Técnica

Esta skill eleva la plantilla de un "prototipo funcional" a un "producto de alta gama". Se debe poner especial atención en cómo se sienten las interacciones al usar la herramienta de OCR y la navegación entre sesiones.
